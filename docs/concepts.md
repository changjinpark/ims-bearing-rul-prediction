# 기초 개념 설명

이 문서는 블로그를 읽다가 낯선 용어가 나왔을 때 빠르게 확인하는 용도입니다.
딥러닝이나 기계 진동 분석을 처음 접하는 독자를 기준으로 설명합니다.

핵심 흐름은 아래와 같습니다.

```text
원본 진동 숫자
-> feature 추출
-> RUL 라벨 생성
-> 모델 학습
-> 실제 RUL과 예측 RUL 비교
```

<h2 id="concept-feature">Feature란?</h2>

Feature는 모델에게 주는 판단 단서입니다.

원본 파일 하나에는 진동 숫자가 약 20,480개 들어 있습니다.
이 숫자 전체를 그대로 보면 의미를 파악하기 어렵습니다.
그래서 원본 진동을 몇 개의 요약 숫자로 바꿉니다.

```text
원본 진동 숫자 20,480개
-> RMS, kurtosis, FFT power 같은 요약값
-> 이 요약값들이 feature
```

건강검진에 비유하면 쉽습니다.

```text
건강 상태를 판단하는 단서:
체온, 혈압, 심박수

베어링 상태를 판단하는 단서:
RMS, kurtosis, crest factor, FFT power
```

이번 프로젝트에서는 딥러닝이 feature를 자동으로 만든 것이 아닙니다.
사전에 정의한 수학/통계/신호처리 공식으로 해석 가능한 feature를 먼저 계산했습니다.

<h2 id="concept-rms">RMS란?</h2>

RMS는 진동이 평균적으로 얼마나 큰지 나타내는 값입니다.

```text
RMS가 커진다
-> 전체 진동 세기가 커진다
-> 베어링 상태가 나빠지고 있을 수 있다
```

예를 들어 조용히 돌아가던 베어링이 점점 크게 흔들리면 RMS가 커질 수 있습니다.
그래서 RMS는 베어링 고장 진단에서 자주 확인하는 기본 feature입니다.

<h2 id="concept-kurtosis">Kurtosis란?</h2>

Kurtosis는 갑자기 튀는 충격이 얼마나 많은지 보는 값입니다.

정상 진동은 비교적 부드럽습니다.
하지만 베어링에 흠집이 생기면 회전 중 특정 순간에 큰 충격이 생길 수 있습니다.

```text
kurtosis가 커진다
-> 중간중간 튀는 큰 진동이 많아졌다
-> 결함 신호일 수 있다
```

RMS가 전체 진동 크기를 보는 값이라면,
kurtosis는 순간적으로 튀는 이상 신호를 보는 값에 가깝습니다.

<h2 id="concept-crest-factor">Crest factor란?</h2>

Crest factor는 평균적인 진동에 비해 가장 큰 순간 진동이 얼마나 튀는지 보는 값입니다.

```text
crest factor = 가장 큰 순간 진동 / RMS
```

쉽게 말하면:

```text
평소에는 조용한데 가끔 크게 튄다
-> crest factor가 커질 수 있다
```

베어링 결함은 순간적인 충격으로 나타날 수 있으므로 crest factor도 고장 징후를 보는 데 사용할 수 있습니다.

<h2 id="concept-fft">FFT란?</h2>

FFT는 시간 기준 진동을 주파수 기준으로 바꾸는 방법입니다.

원본 진동 신호는 이런 질문에 답합니다.

```text
시간이 지나면서 진동값이 어떻게 변했나?
```

FFT 결과는 이런 질문에 답합니다.

```text
어떤 주파수의 진동이 강한가?
```

음악에 비유하면 쉽습니다.

```text
원본 소리:
시간에 따라 흔들리는 파형

FFT:
저음, 중음, 고음 중 어떤 성분이 강한지 분해한 결과
```

베어링에 결함이 생기면 특정 주파수 대역의 진동 에너지가 커질 수 있습니다.
그래서 이번 프로젝트에서는 FFT 결과에서 `fft_total_power`, `band_energy` 같은 주파수 feature를 만들었습니다.

<h2 id="concept-rul">RUL과 라벨이란?</h2>

RUL은 Remaining Useful Life의 약자입니다.

쉽게 말하면:

```text
고장까지 얼마나 남았는가?
```

입니다.

예를 들어 어떤 베어링이 100번째 파일에서 고장났다고 가정하면:

```text
90번째 파일의 RUL = 10
99번째 파일의 RUL = 1
100번째 파일의 RUL = 0
```

라벨은 모델이 맞혀야 하는 정답입니다.
이번 데이터에는 파일별 RUL 정답 CSV가 따로 들어 있지 않았습니다.
그래서 아래 가정으로 직접 라벨을 만들었습니다.

```text
마지막 측정 파일 = 고장 시점
현재 파일의 RUL = 마지막 파일까지 남은 파일 개수
```

즉 마지막 파일은 RUL 0이고, 그 전 파일은 RUL 1입니다.

<h3 id="concept-capped-rul">capped RUL이란?</h3>

초기 정상 구간은 고장까지 너무 많이 남아 있습니다.
예를 들어 첫 파일의 RUL이 983일 수 있습니다.

이 큰 숫자를 그대로 예측하게 하면 모델 학습이 어려워질 수 있습니다.
그래서 너무 큰 RUL은 125로 묶었습니다.

```text
rul_step = 983 -> rul_capped_125 = 125
rul_step = 500 -> rul_capped_125 = 125
rul_step = 30  -> rul_capped_125 = 30
rul_step = 0   -> rul_capped_125 = 0
```

이렇게 하면 모델은 초반 정상 구간의 큰 숫자보다,
고장에 가까워지는 마지막 구간의 변화에 더 집중할 수 있습니다.

<h2 id="concept-gru-lstm">GRU와 LSTM이란?</h2>

GRU와 LSTM은 시간 순서가 있는 데이터를 다루는 딥러닝 모델입니다.

이번 데이터는 파일이 시간 순서대로 쌓여 있습니다.

```text
step 1의 베어링 상태
-> step 2의 베어링 상태
-> step 3의 베어링 상태
-> ...
```

RandomForest나 기본 MLP는 보통 한 시점의 feature 한 줄을 보고 예측합니다.

```text
step 100 feature 한 줄
-> 모델
-> step 100 RUL 예측
```

GRU와 LSTM은 여러 시점의 흐름을 같이 봅니다.

```text
step 71 ~ 100 feature들
-> GRU 또는 LSTM
-> step 100 RUL 예측
```

둘 다 시계열 모델이지만 구조가 조금 다릅니다.

```text
LSTM:
오래 전 정보까지 기억하기 위해 조금 더 복잡한 구조를 가진 모델

GRU:
LSTM보다 구조를 단순화한 모델
```

어느 쪽이 항상 더 좋은 것은 아닙니다.
데이터와 설정에 따라 결과가 달라질 수 있습니다.

<h2 id="concept-baseline-models">Baseline, RandomForest, MLP란?</h2>

Baseline은 비교 기준이라는 뜻입니다.

딥러닝 모델을 만들었을 때 바로 “좋다/나쁘다”를 판단하기 어렵습니다.
그래서 먼저 비교 기준 모델을 만듭니다.

```text
비교 기준 모델보다 좋다
-> 새 모델을 만든 의미가 있다

비교 기준 모델보다 나쁘다
-> 모델이나 데이터 처리를 다시 봐야 한다
```

이번 프로젝트에서는 세 가지 비교 기준 모델을 사용했습니다.

```text
DummyRegressor:
거의 아무것도 배우지 않고 평균값 같은 단순한 값만 예측하는 모델

RandomForestRegressor:
여러 개의 의사결정나무를 사용하는 전통 머신러닝 모델
표 형태 feature 데이터에서 강한 기준 모델로 자주 사용됨

MLPRegressor:
여러 층의 뉴런을 쌓은 기본 신경망 모델
```

이 세 모델이 모든 과제의 정답 조합이라는 뜻은 아닙니다.
이번 프로젝트에서는 쉬운 기준선부터 강한 feature 기반 모델, 간단한 신경망까지 비교하기 위해 선택했습니다.

<h2 id="concept-split-validation">Train/test split과 validation이란?</h2>

모델 학습에서는 데이터를 역할별로 나눕니다.

```text
train:
모델이 공부하는 데이터

validation:
학습 중간에 과적합 여부를 확인하는 데이터

test:
최종 시험 데이터
```

시험에 비유하면 쉽습니다.

```text
train = 교과서와 문제집
validation = 모의고사
test = 기말고사
```

중요한 원칙은 test 데이터로 공부하면 안 된다는 것입니다.
test 데이터를 학습에 섞으면 실제보다 성능이 좋아 보일 수 있습니다.
이것을 데이터 누수라고 부릅니다.

이번 프로젝트의 최종 평가는 실험 단위로 나누었습니다.

```text
train:
1st_bearing3
1st_bearing4
3rd_bearing3

test:
2nd_bearing1
```

즉 `2nd_bearing1`은 모델이 학습 중에는 보지 않고,
마지막 평가에만 사용한 데이터입니다.

<h2 id="concept-metrics">MAE, RMSE, R2란?</h2>

MAE, RMSE, R2는 모델 예측이 얼마나 맞았는지 보는 평가 지표입니다.

MAE는 평균 절대 오차입니다.

```text
실제 RUL과 예측 RUL의 차이를 절댓값으로 바꾼 뒤 평균낸 값
작을수록 좋음
```

예를 들어 실제 RUL이 30인데 모델이 35라고 예측하면 오차는 5입니다.

RMSE는 큰 오차를 더 강하게 벌주는 지표입니다.

```text
큰 실수를 많이 하면 RMSE가 크게 나빠짐
작을수록 좋음
```

R2는 모델이 정답 변화를 얼마나 잘 설명하는지 보는 지표입니다.

```text
1에 가까울수록 좋음
0 근처면 평균 예측과 비슷함
음수면 평균 예측보다 못할 수 있음
```

이번 프로젝트에서는 MAE를 가장 직관적으로 보면 됩니다.

```text
MAE = 4.53
-> 평균적으로 RUL을 약 4.53 step 정도 틀렸다
```
